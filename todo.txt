    - Some actions require confirmation or a target. Therefore, we need to determine how we do that.
    * Example:
        Player chooses to use ability Fireball, which has a BlastAOE and requires a target.
        We let the user select the position with the mouse or arrows keys, using enter or clicking to determine
        the spot.
        Once the spot is selected, we set the player's next action to AbilityAction(..., Fireball(target, ...))

        In order to do this, we need
        - Input Handler to handle the appropriate mouse movement, arrow keys, enter, and click
        - View of the Zone
        - View of the targeting reticule.
    * Example:
        Player must confirm a choice (e.g. walk through a portal)
        User must use Yy (enter) / Nn to confirm or cancel
        In order to do this, we need
        - Input handler to handle confirmation (ConfirmationState)
        - Text prompt added to current view to confirm.
    - Abilities. What kinds are there? What do they do?
        Activated Abilities. Have an optional Cost, optional Cooldown
            e.g.
                Fireball :: Ability
                    Cooldown: 4
                    Effects = [DamageEffect(10d6, Fire)]
                    Target: Position(x, y)
                    AOE: BlastAbility
                    ignoreCaster: false //TODO: Do I want this?
                MageArmor
                    AOE: Point // Caster position
                    Cost: 10 manage
                    Cooldown: 10
                    Effect:
                        BuffEffect (Buff - 10 ac, 5 turns)
        Passive Abilities - different here with feats? Maybe nothing. I think these can be separate.
        Toggled Abilities - abilities that consume some cost and then give a benefit while active
            e.g.
                Quickness
                    AOE: Self
                    Cost: 10 mana
                    Effect:
                        BuffEffect (Buff - +10 agility)
